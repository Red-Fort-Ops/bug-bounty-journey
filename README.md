<span style="color:#ff0000">Welcome</span> to the central repository for **Red Fort Ops'** Bug Bounty initiative. This workspace tracks our structured journey from mastering web security fundamentals to hunting on live Vulnerability Disclosure Programs (VDPs). 

## 🎯 Our Mission
We are a collective of security enthusiasts transitioning into the Bug Bounty ecosystem. Our core philosophy is built on **Manual Testing**. Before relying on automated scanners, we believe in deeply understanding web architecture to uncover complex Business Logic Errors, IDORs, and chain vulnerabilities.

## 🗺️ The Roadmap (Phase-by-Phase)

We are following a highly structured approach based on the 17 learning paths from the **PortSwigger Web Security Academy**:

### ⏳ Phase 1: Core Mechanics (Duration: 6 Weeks)
*Focus: 1 Hour/Day*
Building a rock-solid foundation in web mechanics and server-client interactions.
- [ ] Server-side vulnerabilities
- [ ] File upload vulnerabilities
- [ ] Path traversal
- [ ] Cross-origin resource sharing (CORS)
- [ ] Authentication vulnerabilities
- [ ] Clickjacking (UI redressing)

### 🚀 Phase 2: Intensive Mastery (Durartion: 2 Months)
*Focus: 4-5 Hours/Day*
Deep dive into high-impact vulnerabilities and complex lab scenarios.
- [ ] SQL injection
- [ ] Server-side request forgery (SSRF)
- [ ] Cross-site request forgery (CSRF)
- [ ] API testing & GraphQL API vulnerabilities
- [ ] NoSQL injection & Web cache deception
- [ ] Race conditions & WebSockets vulnerabilities
- [ ] Prototype pollution & Web LLM attacks

### 🌐 Phase 3: Live Hunting
*Focus: Real-world Targets*
Transitioning from vulnerable labs to actual VDPs on platforms like HackerOne and Bugcrowd, focusing purely on manual logic flaws and wide-scope targets.

## 🤝 Team Workflow & Collaboration
While bug bounty is an individual pursuit, Red Fort Ops serves as our **Knowledge Sharing Hub**.
* **Solo Execution:** Members tackle labs and live targets individually to build self-reliance.
* **Collective Growth:** We use GitHub Projects (Kanban Board) to track progress. Bypasses, custom cheat sheets, and new recon techniques are shared here.

## ⚠️ Disclaimer on Write-ups
To respect the PortSwigger Web Security Academy's terms of service, **we do not publish direct solutions or step-by-step walkthroughs of the labs.** Instead, the write-ups in this repository focus on:
* Core concepts learned.
* Custom cheat sheets.
* Methodologies for finding specific bug classes.
